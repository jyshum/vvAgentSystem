Admin Navigation Structure

The admin has 3 levels of depth: Clients list → Client detail → Report editor. Which structure handles this best?

A

Top nav + in-page tabs on detail pages

Keep the flat top nav (Clients · Sign Out). On the Client detail page, add a sticky tab row beneath the heading to jump between Config, Users, Runs, and Reports. Closest to landing page simplicity.

Top nav + breadcrumb trail (recommended)

Nav matches the landing page exactly (Clients link active-highlighted, Sign Out). Below the nav, a slim breadcrumb line shows depth: Clients / ChildSpot / Reports.
