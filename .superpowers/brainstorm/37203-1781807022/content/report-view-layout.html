Report View — Reading Experience

The report has ~9 sections: header, exec summary, KPI grid, competitor table, per-query results, search console, highlights, work completed, priorities. Three ways to frame that long scroll.

A

Editorial document + sticky report bar (recommended)

Keep the single-column document feel — it reads like a VV editorial piece. Add a slim sticky bar just below the nav: ← Dashboard on the left, "Print / PDF" CTA on the right. Previous/next report links at the bottom. On mobile, padding tightens and the title scales down. No structural change, maximum polish.

B

TOC sidebar + scrollable document

Left sidebar shows a sticky table of contents (Executive Summary, KPIs, Competitors, Query Results, Highlights, Work, Priorities). Active section highlights as you scroll. Document scrolls on the right. On mobile, TOC becomes a horizontal pill row you can tap to jump. Good for long reports but adds layout complexity.

C

Full-bleed immersive · sections feel like landing page blocks
